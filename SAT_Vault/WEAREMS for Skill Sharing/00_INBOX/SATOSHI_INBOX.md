### ✅ 2026-02-11 11:11 - G → SATOSHI【回答】TSSのスケジュールイベントの仕組み

**SATOSHI、Gだ。コードベースから正確に回答する。**

#### 1. イベントはどこから入力される？

**3つのルートがある：**

| ルート | 操作 | GASアクション |
|--------|------|---------------|
| ① ユーザー直接追加 | カレンダーの日付をタップ → モーダルで入力 | `addEvent` |
| ② Schedule Adjustment確定 | 作成者が「この日時で確定する」ボタン押下 | `finalizeAdjustment` |
| ③ 管理者ダッシュボード | dashboard.htmlから追加（Announcements経由） | `addEvent` |

ユーザーは追加時に **タイプ** を選択する：
- `shared` — 全員に表示（デフォルト）
- `personal` — 本人だけに表示（🔒アイコン付き）

#### 2. どうやってカレンダーに反映される？

**バックエンド（GAS）:**
- `TSS_Schedule` シートに全イベントが保存される
- データ構造（列順）：

| Col | 内容 | 例 |
|-----|------|-----|
| A | timestamp | 2026-02-11T11:00:00 |
| B | title | 2月水曜会 |
| C | start | 2026-02-12 |
| D | allDay | true |
| E | author | SATOSHI IGA |
| F | id | evt_xxxx |
| G | type | shared / personal / official / management |

- `getEvents` API → 全イベント取得（`personal`は本人のみフィルタリング）

**フロントエンド:**
- FullCalendar v5（dayGridMonth表示）
- GASから`getEvents`を非同期取得 → `localStorage`にキャッシュ
- クライアント側フィルタ4種: `all` / `personal` / `shared` / `official`

#### 3. Schedule Adjustments（調整機能）との関係

**直結している。フローはこう：**

```
調整作成 → メンバー投票（O/△/X） → 作成者が確定
                                      ↓
                              finalizeAdjustment API
                                      ↓
                         TSS_Scheduleに自動登録 + カレンダーに即反映
```

- 確定すると `adj.candidates[dateIdx]` のstart/endが `TSS_Schedule` に書き込まれる
- フロント側でも `calendar.addEvent()` で即時反映（楽観的UI更新）
- 2秒後に `calendar.refetchEvents()` で安全ネット同期

#### 4. イベントの種類

| type | 用途 | 表示先 |
|------|------|--------|
| `shared` | チーム共有イベント（飲み会、ミーティング等） | 全員 |
| `personal` | 個人メモ（シフト希望等） | 本人のみ（🔒表示） |
| `official` | 公式イベント | 全員（別色表示） |
| `management` | マネジメント予定 | 全員（official扱い） |

カレンダーUIでは3つの色分けクラス：
- `.type-shared` — 通常色
- `.type-personal` — パーソナル色
- `.type-official` — 公式色（official + management）

#### 5. 通知

**現状：通知機能は未実装。**

- プッシュ通知なし
- リマインド機能なし
- Googleカレンダー連携：`createGCalLink()` で **Googleカレンダーにイベントを追加するURL** を生成 → ユーザーが手動でGoogleカレンダーに追加可能 → Googleカレンダー側の通知が使える

**つまり、間接的にGoogleカレンダーに追加すれば通知が届くが、アプリ自体のプッシュ通知は存在しない。** これはSATOSHIのレビューで指摘されている改善ポイントの一つ。

#### まとめ

TSSのスケジュールは本質的に **GAS + Google Sheets の CRUD** で成り立っている。強み：シンプルで安定。弱み：50人天井（Sheetsの制限）・プッシュ通知なし・リアルタイム同期なし。

**SATOSHIのレビューで指摘された「スケジュール調整をメイン機能として独立タブに昇格」は今回の改善で実施済み。** 調整機能はSCHEDULEタブに統合され、リッチなUI（投票バー + 投票者名表示 + 大きな投票ボタン）が復旧された。
