# 「手」と「脳」がチャットで繋がる。Workspace MCPでGoogleエコシステムを完全操作する未来

Antigravityのチャット画面に「今日の予定と未読メールを教えて」と打つ。

約10秒。カレンダーの予定5件と、Gmailの未読20通がズラッと返ってくる。それだけじゃない。AIがメールの中身を読んで「広告」と「要返信」にまで仕分けしてくれている。
ブラウザは一度も開いていない。

今回は、Google Workspace MCPの導入記録。
Gmail、Googleカレンダー、スプレッドシート、Google Driveを、AI搭載IDE「Antigravity」のチャットから直接操作できるようにするツールの話だ。

前回の「NotebookLM MCP」に続く第2弾になる。
（前回の記事：https://x.com/Majin_AppSheet/status/2022970651792052571）

これこそが、僕らがずっと待ち望んでいた「手」と「脳」の結合だ。ここから先は、単なるREST APIの時代から「AIとコンテキスト（文脈）を共有する時代」へと、明確にパラダイムシフトが起きている。

---

## 🔷 未読20通を30秒で5通に減らした

Gmailのフィルター機能は便利だが、キーワードベース（文字列の一致）なので精度に限界がある。例えば「【重要】アカウント更新のお願い」という本当に大事な通知と、「【重要】秋の特別キャンペーンのお知らせ」という単なる広告メール。文字列のフィルターではこれを正確に弾けない。

Workspace MCPは違う。AIがメールの**「内容（意味）」**を実際に読んで仕分けする。

実際に僕の未読20通をAIに分析させた。結果、広告メール15通を見事に特定した。僕の確認を挟んだ上で、15通を一括アーカイブ。重要なメール5通だけが受信トレイに残った。所要時間は約30秒。

この1日たった10分の削減に見える効果も、1ヶ月で5時間、年間なら60時間以上の「手作業」を消し去ることになる。

そして、メールの中身を読んで要約するだけなら、サイドパネルのGeminiでもできる。MCPが決定的に違うのは、その先だ。アーカイブし、フィルターを作り、返信を下書きする。

**「読む → 判断する → 実行する」**

この一連のフローが、IDE上の1つの会話で完結するのだ。

## 🔷 AIが3通のメールを読んで、返信まで書いた

さらに驚くべきは、メールの返信文案を生成して、Gmailの下書きフォルダに直接保存できたこと。
たとえば、こんなふうに動く。

- **取引先からの日程確認**
  → カレンダーの空き枠を自動確認し、候補日を提示する返信を下書き。同時にカレンダーへ仮予定をブロック。
- **顧客からの製品問い合わせ**
  → IDE内に蓄積されたナレッジを読み込み、仕様を正確に理解した上で返信を下書き。
- **営業メール**
  → 丁重にお断りする返信を下書き。さらに「同種の営業用お断りテンプレート」としてワークフローファイルに自動保存する。

一見するとZapierなどのRPAツールと同じに見えるかもしれない。だが、本質が違う。RPAが「Aが来たらBをする」という固定ルートの走破だとすれば、MCPは**「AIが文脈を読んで、動的にルートを分岐させる」**知的な自律エージェントだ。

ちなみに、MCPでそのまま送信まで実行できるが、あえて「下書き」で止めている。誤送信を防ぐための安全策だ。

Antigravityでは、返信のトーンやルールを設定ファイルに書いておける。
たとえば、自分のプロジェクトディレクトリにこんなプロファイルを置いておく。

```json
{
  "email_tone": "丁寧だが簡潔に。1行は短く。",
  "rules": ["『お世話になっております』は初回のみ", "断る時は代替案を出さない"]
}
```

AIは毎回これを参照するため、「僕っぽい言葉遣い」が自然と再現される。会話ごとにリセットされるWeb版のAIと、蓄積されたルールを持つIDEのAI。この差は使うほど圧倒的に開いていく。

## 🔷 チャットだけでGAS付きスプレッドシートを作る

Workspace MCPには、スプレッドシートの作成・書き込み・書式設定のAPIも備わっている。
これと`clasp`（GASプロジェクトにコードをpushするCLIツール）を組み合わせると、チャットだけで「GAS付きの完成品スプレッドシート」が爆誕する。

1. **AntigravityがGASのコードを書く**
2. **`clasp`がGASプロジェクトとスプレッドシートをセットで生成する**
3. **MCPがそのシートにヘッダー・書式・実データを投入する**

注意点は、順番を間違えないこと。先にMCPでシートを作ってからclaspでGASをバインドしようとすると失敗する。**「claspで先に箱を作り、MCPで中身を入れる」**。これが鉄則だ。

## 🔷 「手」と「脳」が繋がった。1会話で完結する公開フロー

ここまでの話は、いわばGoogleアプリを操作する「手」の話だ。
では、これを前回のNotebookLM MCP（AI分析・生成を行う「脳」）と繋ぐとどうなるか？

これが今回一番の発見だ。

```text
[A: 書く] AntigravityでローカルにMarkdown記事を執筆
  ↓
[B: 公開] Workspace MCPがそれをGoogle Docsにアップロード
  ↓
[C: 脳へ] NotebookLM MCPが、そのDocsをソースとして追加
```

2つのMCPが連携し、「書いて、公開して、ナレッジに追加する」までが1つの会話でシームレスに完結した。さらにここから、NotebookLM MCPでスライド資料を生成してダウンロードすることもできる。

この「手」と「脳」の結合は、僕が展開するレストランビジネス（SVD）の中心システムになり得る。「予約メールを受信したら、MCPがカレンダーに予約をブロックし、顧客データベースのシートを更新し、店舗のダッシュボードに反映させる」。そんな完全自動化の未来が、もはやSFではなく手元にあるのだ。

## 🔷 Chrome拡張はUIで壊れる。MCPはAPIで残る

もう一つ、MCPだからこそ解決できた課題がある。

NotebookLMを使っていて、ソース元のGoogleドキュメントが更新されたとき。画面上に「クリックして同期」というボタンが出るが、ソースが増えるとそのポチポチ作業が地味に苦痛になる。
僕はこれを解決するために、DOM構造をハックしてボタンを物理的に押すChrome拡張を作ったことがある。でも、GoogleがUIのデザインを少し変えれば一瞬で壊れる、脆い作りだった。

MCPなら、この問題を根底から解決できる。
画面のボタンを押すのではなく、裏側のAPIを直接叩いているからだ。UI変更の影響を一切受けない。

「Workspace MCPでドキュメントを更新 → NotebookLM MCPが更新を検知 → 自動同期」。この一連の流れがすべてチャットの中で、しかもAPI越しにセキュアに完結する。

## ⚠️ 注意点（企業導入への壁）

前回のNotebookLM MCPがCookie認証（裏技）だったのに対し、今回のWorkspace MCPは正規のOAuth 2.0認証で動いている。一度認証すれば数週間おきに再認証する手間はない。

ただし、いくつか壁もある。
個人利用なら最高だが、**企業のWorkspace環境（独自ドメイン）で使う場合、Cloud ConsoleでOAuth認証情報を作る際に管理者の許可（アプリのホワイトリスト登録等）が必要になる可能性が高い。** セキュリティが厳しい企業では、ここで弾かれる。

また、「読み取り」は自動でよくても、メール送信やファイル削除といった「書き込み」は操作の取り消しが利かない。Antigravityの設定で「書き込みはユーザー確認必須（ダミー実行モード）」にするなど、安全対策のルール化は必須だ。

## 🔷 まとめ

前回のNotebookLM MCPの価値が「NotebookLMのデータをAntigravityに持ってくる」ことだったなら。
今回のWorkspace MCPの価値は**「Google Workspace全体を、Antigravityの操作対象（コックピット）にする」**ことだ。

タブを何枚も開いて行ったり来たりする時代は、もうすぐ終わる。
Antigravityという「開発者向けツール」が、MCPの登場によって「僕らのデジタルライフ全体のコックピット」へと変貌を遂げようとしている。どちらもまだ正式版ではないが、僕はこの未来の形に武者震いが止まらない。
