# Definition of Deliciousness — 美味しさの定義 🍽️✨

SVDが開発した独自のレストラン評価フレームワーク。「美味しさ」を定量化し、解像度を高めることで本質的な美味しさを追求する。

## Core Rules

### 美味しさの方程式

**美味しさは「味」だけではない。** 複数の要素が重なり合い、掛け合わされることで生まれる**複合的な体験的価値**。

### 1. 美味しさスコア A（500点満点）

```
A = {[(FD① + FD②) × (SV③ + SV④)] + ⑤場所} + ⑥ともに過ごす人
```

#### 料理 (Food: FD)
- **FD① 料理のあじわい**（10点満点）: 五味の調和、味、風味（香り）、食感、供出温度、コスト構造、技術的完成度
- **FD② 料理の設え**（10点満点）: 第一印象（盛り付け）、食材の質、器、触感、独自性・創造性、食べやすさ

#### サービス (Service: SV)
- **SV③ 空間づくり**（10点満点）: 第一印象（雰囲気）、導線、演出、照明、室温・湿度、調度品、BGM・雑音
- **SV④ オペレーション**（10点満点）: 技術・知識、タイミング、ハウスルール、システム、サービス力、組織力

#### その他 (Others) — 定数
- **⑤ 場所**（50点満点）: 景色、利便性、トリップ価値、可視性、ブランド認知度
- **⑥ ともに過ごす人**（50点固定）: 顧客自身が選択する要素（店側では変えられない）

### 2. 期待値スコア B（上限1.5ポイント）

```
B = ⑦料理期待値 + ⑧場所期待値 + ⑨ブランド期待値
```

- ⑦ 料理期待値（0.4〜0.6pt）
- ⑧ 場所期待値（0.4〜0.6pt）
- ⑨ ブランド期待値（0.1〜0.3pt）

**※期待値が低いほど高ポイント** — 期待を超えやすい

### 3. レストラン満足度 C（最大750Pt）

```
C = A × B
```

### 特に印象のない要素の基準値

**特に印象のない場合は 6.0 とする**（これが中央値）

## Anti-patterns

- ❌ 「味」だけで美味しさを語ってはいけない（味は全体の一要素）
- ❌ FD（料理）とSV（サービス）は**足し算ではなく掛け算** — どちらかがゼロなら全体もゼロ
- ❌ ⑤場所と⑥人は**定数**として扱う（店が変えられない要素）
- ❌ 期待値を無視してはいけない — 無名店の感動 > 有名店の当然
- ❌ 「なんとなく美味しかった」で終わらせない — 必ず数値化する

## Examples

### SAT's An Di Review (記入例)
```
Restaurant: An Di（モダンベトナミーズ／東京・外苑前）
FD① = 9.17（レフェルヴェソンスで修行したシェフの技術）
FD② = 8.33（前衛的ジャンルでの統一感）
SV③ = 8.29（オープンキッチンの臨場感）
SV④ = 9.31（小林氏の素晴らしいサービス）
場所 = 45（外苑前、少し迷う立地）
人   = 50

A = {[(9.17+8.33) × (8.29+9.31)] + 45} + 50
  = {[17.50 × 17.60] + 45} + 50
  = {308.0 + 45} + 50
  = 402.97

期待値 B = 1.10
満足度 C = 402.97 × 1.10 = 443.27
→ 🔥 HIGH-SATISFACTION
```

## Key Insight: SVDの戦略的優位性

SVDの4レストランは**⑤場所**で圧倒的な強みを持つ（藻岩山の夜景、大倉山のジャンプ台、テレビ塔の眺望）。
さらに**期待値が低い**（観光地レストラン = 期待されにくい）ため、品質を上げれば**期待を大きく超える満足度**を実現できる。
