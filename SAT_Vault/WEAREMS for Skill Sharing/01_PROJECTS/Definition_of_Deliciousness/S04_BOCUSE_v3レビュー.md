## セクション B：BV軸の実装品質 — 世界基準で通用するか

### 構造的位置付け

BVはルーブリック v2.0 に**独立セクションとして新設**。方程式 v3.0 本体には組み込まれていない。

**これは何を意味するか？** BVの採点基準は存在するが、**A値の計算には反映されない**。FD①②とSV③④の4カテゴリでCoreを算出する構造は不変。BVはルーブリック上の「採点はするが方程式に入らない」オルファン項目になっている。

### 問題提起：BVは方程式に入るべきか？

**世界基準での比較：**

| 評価体系 | ビバレッジの扱い |
|---------|---------------|
| ミシュラン | 料理評価とは別にソムリエ賞あり（独立評価） |
| World's 50 Best | 「dining experience全体」に内包（独立項目なし） |
| Gault & Millau | ワインリストを独立評価（料理と別スコア） |
| La Liste | ビバレッジを総合スコアに含む |
| **SVD v3.0** | **ルーブリックに存在するが、A値（方程式）には不参加** |

SVDの現在の構造は**ミシュランとGault & Millauのハイブリッド**に近い。採点基準はあるが方程式には入れない——これは「ビバレッジは美味しさの本質ではなく、体験の付加価値である」という哲学的判断として読める。

**私の見解：** BVを方程式に入れないことは**現時点では許容可能**。理由：

1. SVDの方程式はまだ実運用前。まずFD+SVの4軸で安定稼働させてからBVを検討するのが現実的。
2. BVを入れるとCore構造が変わる。(FD① + FD② + BV) × (SV③ + SV④) か？ (FD① + FD②) × (SV③ + SV④) × BV係数 か？ 構造設計が必要。
3. ルーブリックで採点し、PDCAサイクルで追跡するだけでも、BVの品質管理は可能。

**ただし、将来的にはBVを方程式に組み込む道筋を示すべき。** 具体的には：
- Phase 1（現在）：ルーブリックで採点のみ。方程式外。
- Phase 2：BVをFD軸に統合（FD③として）、またはBV係数（×0.90〜1.10）として乗算。
- Phase 3：ペアリング体験をFD×BVの相互作用として設計。

## セクション C：v3.0で新たに生まれた問題点

---

### C-1：P-Scoreの測定設計が粗い

P-Scoreは哲学的に極めて重要だ。しかし現在の定義は：

```
P-Score = 定期サーベイ（四半期ごと）
  ├── 「今の仕事に誇りを感じていますか」
  ├── 「家族にこの仕事を語りたいですか」
  └── 「SVDで働いていることを嬉しく思いますか」
```

**問題点：**
1. **3問で何を測るのか。** スケールは？ 5段階リッカートか？ 10段階か？ YES/NOか？
2. **匿名性の担保。** SATが直接結果を見る構造では、正直な回答が得られない。第三者集計が必要。
3. **P-Score → A値との関係。** P-Scoreが低いとA値にどう影響するのか？因果関係の仮説がない。「P-Scoreが高ければA値も上がる」は直感的に正しいが、測定で確認すべき。
4. **3問の質問が似すぎている。** 「誇り」「語りたい」「嬉しい」は実質的に同じ情動を異なる角度から聞いているだけ。**従業員エンゲージメントの学術的知見（Gallup Q12等）を参照し、多軸化すべき。**

### C-2：FD①のサブ項目数膨張
