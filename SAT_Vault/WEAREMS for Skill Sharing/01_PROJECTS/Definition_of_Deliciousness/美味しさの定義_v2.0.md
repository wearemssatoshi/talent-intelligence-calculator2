## 第1章: 方程式 v2.0

### v1.0の問題点（BOCUSE指摘）

```
❌ v1.0: A = {[(FD①+FD②) × (SV③+SV④)] + ⑤場所} + ⑥人
問題: FD×SV=0でもA=100になる。「掛け算の哲学」と矛盾。
```

### v2.0 構造

```
A（美味しさスコア 500pt）= Core × Place × Companion

Core（体験品質 400pt）= (FD① + FD②) × (SV③ + SV④)
  ├── FD① 料理の味わい（10pt）
  ├── FD② 料理の設え（10pt）
  ├── SV③ 空間づくり（10pt）
  └── SV④ オペレーション（10pt）

Place（場所係数 ×0.80〜1.25）
  └── SVDの場所性を係数で反映（定数ではなくアイデンティティ係数）

Companion（共食係数 ×0.90〜1.10）
  └── LTCシーン別の体験増幅（通常=1.0、記念日=1.05、プロポーズ=1.10）
```

### なぜv2.0か

| 項目 | v1.0 | v2.0 |
|------|------|------|
| ⑤⑥の扱い | 足し算（矛盾） | **係数（掛け算で統一）** |
| Core=0の場合 | A=100（矛盾） | **A=0（哲学通り）** |
| ⑥の固定問題 | 50pt固定 | **LTCシーン別の係数** |
| 場所の意味 | 物理的立地評価 | **アイデンティティ係数** |

### v2.0 計算例: An Di

```
FD①=9.17, FD②=8.33, SV③=8.29, SV④=9.31
Core = (9.17+8.33) × (8.29+9.31) = 17.50 × 17.60 = 308.0
Place = ×1.00（外苑前、標準的な都市型）
Companion = ×1.00（通常の食事）

A = 308.0 × 1.00 × 1.00 = 308.0

B（期待値）= 1.10
C（満足度）= 308.0 × 1.10 = 338.8
```

### v2.0 計算例: JW-D（SVD The Jewels ディナー）

```
FD①=8.0, FD②=8.0, SV③=6.5, SV④=6.5
Core = 16.0 × 13.0 = 208.0
Place = ×1.25（藻岩山夜景、SVD最高の場所性）
Companion = ×1.05（記念日ディナー）

A = 208.0 × 1.25 × 1.05 = 273.0

B（期待値）= 1.20（観光地レストラン = 低期待値）
C（満足度）= 273.0 × 1.20 = 327.6
```

## 第2章: 期待値 B（上限1.5）— v2.0

```
B = ⑦料理期待ギャップ + ⑧場所期待ギャップ + ⑨ブランド期待ギャップ
```

### v2.0 変更点: 「期待値」→「期待ギャップ」

BOCUSEの指摘「成功するほどスコアが下がる自己矛盾」を解消。

| | v1.0 | v2.0 |
|---|------|------|
| 定義 | 期待値の逆数 | **期待と実力の差（ギャップ）** |
| 成功時 | ❌ B↓（不利に） | ✅ A↑とB維持が両立可能 |
| 測定 | 主観的 | **A値の成長速度 vs 認知度の成長速度** |

**原則**: A値がブランド認知に先行して成長し続ければ、B値は維持される。

