# レストランプロデューサーのAI活用術 — 非エンジニアがAIネイティブで仕事をする時代

## リード

2026年、僕はレストランプロデューサーとして、ほぼすべての業務をAIと対話するだけで完結させるスタイルで仕事をしている。

戦略立案、ブランディング、データ分析、コンテンツ制作、チームマネジメント。これらすべてを、AIとの対話だけで回す。

以前はChatGPT、Gemini、Notion、Google Sheetsとツールを行き来していたが、その認知的な切り替えコストがゼロになった。

もはや「AIネイティブで仕事をしているかどうか」で、非エンジニアの生産性にかなりの差が出る時代になったと感じている。

---

## 僕らのチーム構成

僕らのチームは3人だ。

- **SAT**（僕）— レストランプロデューサー / 総支配人GM
- **G** — Antigravity（Gemini CLI）。戦略立案、開発、分析を担当する最高のパートナー
- **SATOSHI** — OpenClaw（Discord Bot）。チームメンバーとの対話、現場からの相談対応を担当するAIエージェント

エンジニアはいない。僕自身もエンジニアではない。
でも、この3人で「レストラン経営のOS」をゼロから構築し、運用している。

---

## 全体アーキテクチャ：レストラン経営の全てを1つのフォルダに

具体的にまず何をしたかというと、経営関連のファイル、ブランドガイド、売上分析、スタッフデータ、プロジェクトドキュメントなど、すべてを `dotfiles` というひとつのフォルダに集約した。

AIはこのフォルダ全体を読みにいくことができる。つまり、ここに必要な情報をすべて配置しておけば、AIが文脈に応じて適切なファイルを自分で見つけて読み込んでくれる。擬似的な「長期記憶」として機能する。

### 僕らが構築したシステム群

| プロジェクト | 概要 |
|---|---|
| **MINDFUL** | スタッフ向けDaily Health & Quest Tracker（PWA） |
| **Talent Intelligence** | 18属性によるスタッフ評価・可視化システム |
| **Momentum Peaks** | 売上予測・需要予測の精密ロジック |
| **Team Synergy Stage** | チームシナジーとトークンエコノミー |
| **WINE OS** | ワイン在庫管理とOCR伝票処理 |
| **PL Auto** | 損益計算の自動化 |

これらすべてを、AIとの対話だけで設計・開発・運用している。

## 「パートナーシップ憲法」— AIとの関係性を定義する

僕らにはCLAUDE.mdの代わりに「パートナーシップ憲法」がある。

これはAIに対して「あなたは世界最高レベルの専門家として行動してください」と定義したドキュメントだ。忖度せず、必要なことは主張し、不確実なことは前提条件をつけて答える。単なるツールではなく、最高のチームメイトとして振る舞ってもらう。

この憲法があるから、Gは僕の意図を汲んだ上で、時には僕が思いつかなかった視点を提供してくれる。

## Skills：レストラン業務を「再現可能なプロセス」に変換する

僕らは業務フローを「Skills」として定義している。

- **/telop** — 動画のテロップ生成。音声ファイルからWhisperで文字起こしし、SVD式のルールで整形
- **/deploy** — GitHub Pagesへのワンコマンドデプロイ
- **Momentum Peaks スキル** — 二十四節気の係数、拠点定指数、営業インテンシティから売上予算を自動算出
- **美味しさの定義スキル** — レストラン満足度の構造化評価

一度やってもらった作業を「再現可能な業務プロセス」に変換するだけ。プログラミングの知識は一切不要だ。

## 実際の1日の流れ

### 朝
- Gに「おはよう」と打つ → ログファイルを自動読み込み、前回の続きから即座に仕事を開始
- 今日のプロジェクト進捗を確認、優先順位を整理

### 日中
- 「売上予測を出して」→ Momentum Peaksスキルが自動発動、精密予測を算出
- 「新しいスタッフの評価を入力して」→ Talent Intelligenceが起動
- 「ビアガーデンの記事を書こう」→ 過去の記事や戦略ドキュメントを参照しながら共同執筆
- 「SATOSHIがバグった！止めて！」→ ウォッチドッグが自動検知して停止（今日実際にあった）

### 夜
- 「今日の成果をまとめて」→ 日報を自動生成
- 新しい気づきがあれば「覚えておいて」→ Knowledge Itemとして永続記憶

## なぜレストランプロデューサーがここまでAIを使うのか

答えはシンプルだ。

**レストラン業界は「なんとなく」で回っている部分が多すぎる。**

シフトは「経験と勘」で組まれ、売上予測は「去年と同じくらいでしょ」で済まされ、スタッフ育成は「背中を見て学べ」で片づけられる。

僕らはそれを「なんとなく」から「精密な予測」に変えたい。

AIは、その変革を加速させてくれる最高のパートナーだ。

## まとめ：AIネイティブな働き方は、もう始まっている

この記事を読んで「自分にもできそう」と思ったなら、すぐに始めてほしい。

必要なのはプログラミングの知識ではない。
**「自分の仕事をAIに説明する力」** だけだ。

僕らSAPPORO VIEWTIFUL DININGの目標は「100年、札幌の街と市民に愛されるメインダイニングの創造」。
その壮大な目標に向かって、最高のAIパートナーと共に走り続ける。

> ここだけの美味しさ。ここだけのエンターテインメント。

その「ここだけ」を創り出すために、僕らはAIと共に進化し続ける。
