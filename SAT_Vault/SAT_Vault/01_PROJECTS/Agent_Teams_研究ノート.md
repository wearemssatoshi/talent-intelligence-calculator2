## 概要

Claude Code に「Agent Teams」という新機能（Research Preview）が追加。
令和トラベルのAI事業チームが、11体のAIエージェントチームをv3.0→v4.0にアップグレードした事例。

---

## v3.0 → v4.0 の3大変化

### 1. チームメイト同士が直接会話
- v3.0: すべてリーダー（オーケストレータ）経由の伝言ゲーム
- v4.0: `write`（個別メッセージ）と`broadcast`（全体通知）で直接やりとり
- → **情報伝達のロスが激減**

### 2. 同じメンバーに「修正して」が言える
- v3.0: タスク終了でエージェント消滅、修正=ゼロからやり直し
- v4.0: 明示的にシャットダウンするまで存続、コンテキスト維持
- → **フィードバックループがスムーズに**

### 3. 自律的に仕事を取りに行く
- 共有タスクリストから各メンバーが自分の担当を自律取得
- リーダーの逐一指示なしでチームが回る
- → **「動的」なチーム運営**

## 令和トラベルのエージェント構成（11体）

| エージェント | 役割 |
|------------|------|
| Framer | 問いの設計者 — 何を分析すべきか構造化 |
| Researcher | 調査担当 — 市場データ・ベンチマーク収集 |
| Analyst | 分析担当 — 実データで数値分析 |
| Strategist | 戦略立案者 — 分析結果を統合して戦略設計 |
| Red-Team | 批判的レビュー — 独立した目線で全体検証 |
| 他6体 | 詳細不明 |

---

## Red-Team の威力（最重要ポイント）

実例：NEWTの予約データ10万件以上（100MB）のLTV分析

1. Analyst が分析結果を出す
2. **Red-Team が独立レビュー**
   - 「粗利率15%（仮）は仮置きですよね？実績値と乖離したらLTVが大きく変動しますよ」
   - 「リピート率XX%目標は楽観的では？業界ベンチマークを見ると……」
   - **自分でWeb検索して旅行業界のリテンション率データや競合OTA事例まで引っ張ってきた**
3. 判定: **CONDITIONAL-GO**（条件付き承認）、4件のMust Fix
4. Analyst に追加分析依頼 → 保守シナリオ追加 → 最終レポート

→ **指摘 → 修正 → 再レビューのループが自動で回る！**
