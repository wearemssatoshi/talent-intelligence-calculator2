# 美味しさ方程式 詳細解説

## 構造

```
C（レストラン満足度）= A（美味しさ）× B（期待値）

A = {[(FD① + FD②) × (SV③ + SV④)] + ⑤} + ⑥

  FD① 料理の味わい（10点満点）
    ├── 五味の調和
    ├── 味
    ├── 風味（香り）
    ├── 食感
    ├── 供出温度
    ├── コスト構造
    └── 技術的完成度

  FD② 料理の設え（10点満点）
    ├── 第一印象（盛り付け）
    ├── 食材の質
    ├── 器
    ├── 触感
    ├── 独自性・創造性
    └── 食べやすさ

  SV③ 空間づくり（10点満点）
    ├── 第一印象（雰囲気）
    ├── 導線
    ├── 演出
    ├── 照明
    ├── 室温・湿度
    ├── 調度品
    └── BGM・雑音

  SV④ オペレーション（10点満点）
    ├── 技術・知識
    ├── タイミング
    ├── ハウスルール
    ├── システム
    ├── サービス力
    └── 組織力

  ⑤ 場所（50点満点）— 定数
    ├── 景色
    ├── 利便性
    ├── トリップ価値
    ├── 可視性
    └── ブランド認知度（場所）

  ⑥ ともに過ごす人（50点固定）— 定数
    └── 顧客自身が選択（店側不可変）

B = ⑦ + ⑧ + ⑨（上限1.5）
  ⑦ 料理期待値（0.4〜0.6）
  ⑧ 場所期待値（0.4〜0.6）
  ⑨ ブランド期待値（0.1〜0.3）
  ※期待値が低いほど高ポイント
```

## 数学的性質

### なぜ掛け算か？

- FDとSVは**足し算ではなく掛け算**
- 料理が10点満点でもサービスが0なら、体験全体が崩壊する
- 逆に、両方が適度でも掛け合わせで大きな数字になる

### 最大値の計算

```
A_max = {[(10+10) × (10+10)] + 50} + 50
      = {[20 × 20] + 50} + 50
      = {400 + 50} + 50
      = 500

B_max = 0.6 + 0.6 + 0.3 = 1.5

C_max = 500 × 1.5 = 750（理論最大値）
```

> 750ptを達成するには「無名で期待値ゼロの店が全要素10点満点」という矛盾的状況が必要。実質的にC=500前後が現実的な天井。

### SVDの戦略的優位性（期待値の力）

```
有名店:    A=365 × B=0.9 = C=329（期待に応えて当然）
SVD(JW):  A=308 × B=1.2 = C=370（期待を超えて感動）
```

**期待値の差(×0.3)が美味しさスコアの差(57pt)を逆転させている。**
