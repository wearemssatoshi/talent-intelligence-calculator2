# Ask Maps × AI Mode Canvas — 北海道レストランへの影響分析

> **日付**: 2026-03-14
> **トリガー**: Google Maps「Ask Maps」リリース + AI Mode Canvas 全米展開
> **対象**: SVD全4店舗 + 赤レンガテラス出店計画

---

## 1. Ask Maps がもたらすパラダイムシフト

### 従来の検索行動
「札幌 ディナー おしゃれ」→ リスト10件 → 口コミ比較 → 決定

### Ask Maps 以後
「札幌駅から徒歩10分で、夜景が見えて、記念日に使えるフレンチは？」
→ **Geminiが厳選1〜3件の回答を生成** → 即予約

### SVDへの影響評価

| 変化 | 影響度 | SVDへの影響 |
|------|:------:|-----------|
| 会話型クエリ | ★★★ | 「眺望」「記念日」等の体験ベースが重要に。SVDのコアコンピタンスに直結 |
| AI推薦の一本化 | ★★★ | 10件リスト → 厳選1〜3件。選ばれる側と選ばれない側の格差拡大 |
| 構造化データ依存 | ★★★ | GBPの情報がAIの判断材料。不完全 = AIに認識されない |
| インバウンド × 多言語 | ★★☆ | 外国人が母国語で質問 → 英語GBPがなければ推薦対象外 |

### SVDのポジション
```
                GBP最適化レベル
             低 ──────────── 高
        ┌──────────┬──────────┐
 体験 高│ 機会損失  │ ★ SVD  │
 価値   │          │ 勝者圏  │
        ├──────────┼──────────┤
     低 │  淘汰    │ 一時的  │
        │  ゾーン   │ 優位   │
        └──────────┴──────────┘
```

## 2. AI Mode Canvas の影響

検索 = 旅行計画ツールになる。

「北海道3泊4日の旅行プランを作って。ディナーは毎晩違うジャンルで。」
→ Canvasが旅程表を自動生成 → レストランが組み込まれる

**個別店舗の検索行為自体が消える。旅程の1ピースとして選ばれるかどうか** が商売の分かれ目。

- Geminiアプリ **月間7.5億ユーザー**
- 北海道を検索するインバウンド客の多くがこのエコシステム内にいる

## 3. 即時アクション

→ [[GBP最適化ワークフロー]] 参照（dotfiles/.agent/workflows/gbp-optimization.md）

1. 全店舗のGBP情報を **英語・中国語・韓国語** で完全整備
2. 体験キーワード（夜景、記念日、大通公園ビュー、藻岩山）をGBP説明文に組み込む
3. メニュー構造化データ（価格帯・コース・アレルギー対応）を最新化
4. 高品質写真を定期的に更新（AIは画像認識も使う）
5. Google予約 / OpenTable連携の検討

## 4. 競合優位性

北海道の多くの飲食店はGBPを「放置状態」にしている。Ask Maps普及時に **GBPを戦略的に最適化した店だけがAI推薦に乗る** 世界が来る。SVDが先手を打てば、札幌のレストラン検索において不動のポジションを確保できる。

これは「アジアベストレストランTOP100」戦略とも合致する動き。デジタルプレゼンスの強化は「マーケティング施策」ではなく **「存在するかしないか」** の問題。
