# /buzz — 業界ニュース → Discordプッシュ

> daily-news-report スキルでニュースを収集し、
> 要約をDiscord Webhookで通知するワークフロー。

## 手順

### 1. daily-news-reportスキルを読み込む

```
スキルファイルを読む: .agent/skills/daily-news-report/SKILL.md
```

### 1-b. 飲食SaaS・ホテル業界・SVD戦略向けのWeb検索を実行する

以下のクエリで最新のWeb検索を実行し、情報を収集する。
1. `TableCheck ダイニー 飲食 SaaS ニュース 2026年3月`
2. `AI レストラン トレンド 世界 2026年3月`
3. `ホテル AI ホスピタリティ トレンド 2026年3月`
4. `タイミー スキマバイト 飲食 トレンド 2026年3月`
5. `外食産業 顧客体験 向上 AI 最新事例 世界 2026年`
6. `レストラン 新しいビジネスモデル 成功事例 2026年`


### 2. ニュースレポートを生成する

daily-news-reportスキルの情報と、ステップ1-bのWeb検索の情報を**用途別に3つのMarkdownレポート**として作成する。

出力先(Tech): `NewsReport/YYYY-MM-DD-news-report.md`
出力先(Food): `NewsReport/YYYY-MM-DD-restaurant-news.md`
出力先(SVD戦略): `NewsReport/YYYY-MM-DD-svd-trends.md`

daily-news-report スキルの手順に従い、ニュースを収集・フィルタリング・レポート生成する。

### 3. レポートからDiscord用サマリーを作成する

生成されたレポートからTOP 5記事を抽出し、以下のフォーマットでDiscordメッセージを作成:

```
📰 **Daily Buzz — YYYY/MM/DD**

🔥 **1. [タイトル]**
> 要約（1-2行）
> 🔗 URL

🔥 **2. [タイトル]**
> 要約（1-2行）
> 🔗 URL

...（TOP 5まで）

━━━━━━━━━━━━━━━━━━
_Generated by SVD Intelligence_
```

### 5. 古いレポートの自動消滅（クリーンアップ）

3日以上経過したレポートファイルは自動的に削除し、情報を新鮮に保つ。

```bash
find /Users/satoshiiga/dotfiles/NewsReport -name "*-news-report.md" -type f -mtime +3 -exec rm {} \;
find /Users/satoshiiga/dotfiles/NewsReport -name "*-restaurant-news.md" -type f -mtime +3 -exec rm {} \;
find /Users/satoshiiga/dotfiles/NewsReport -name "*-svd-trends.md" -type f -mtime +3 -exec rm {} \;
```

### 6. 完了確認

- 新しいレポートが `NewsReport/` に保存されたことを確認
- 古いレポートが削除されたことを確認
- Discordにメッセージが送信されたことを確認
- SATに完了を通知
