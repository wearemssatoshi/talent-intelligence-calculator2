# Definition of Deliciousness v3.0 — 美味しさの定義 🍽️✨

> **「北海道の海と大地が育む美味しさを、100年先のSAPPOROへ届ける。**
> **なぜなら僕らはSAPPORO VIEWTIFUL DININGだから。」**
> — SAT（伊賀智史）

> **「僕のじゃなくて、この街のなんだよ。」** — SAT

SVDが開発した独自のレストラン評価フレームワーク。「美味しさ」を定量化し、解像度を高めることで本質的な美味しさを追求する。

## Core Rules

### 美味しさの方程式 v3.0

**美味しさは「味」だけではない。** 複数の要素が重なり合い、掛け合わされることで生まれる**複合的な体験的価値**。

### 1. 美味しさスコア A（最大550pt）

```
A = Core × Place × Companion
```

#### Core（体験品質 最大400pt）= (FD① + FD②) × (SV③ + SV④)

##### 料理 (Food: FD)
- **FD① 料理のあじわい**（10点満点）: 五味の調和、風味（香り）、食感、供出温度、火入れの精度、技術的完成度
- **FD② 料理の設え**（10点満点）: 第一印象（盛り付け）、食材の質、独自性・創造性

##### ビバレッジ (Beverage: BV) ✦ NEW
- **BV ビバレッジ**（10点満点）: ワイン・ビバレッジセレクション、ノンアルコール・お茶
- **BVはFD②を補完する外部評価軸。Core計算には含めず、ルーブリックで独立採点し改善指標として運用する。**

##### サービス (Service: SV)
- **SV③ 空間づくり**（10点満点）: 第一印象（雰囲気）、照明、BGM・音環境
- **SV④ オペレーション**（10点満点）: 技術・知識、タイミング、サービス力、組織力、Welcome&Farewell、Recovery

#### Place（場所係数 ×0.80〜1.25）
SVDのアイデンティティそのもの。VIEWTIFUL = VIEW + BEAUTIFUL。

| Place特性 | 係数 | SVD例 |
|-----------|------|-------|
| 圧倒的景観体験 | 1.25 | 藻岩山夜景 |
| 象徴的ランドマーク | 1.15 | テレビ塔、大倉山 |
| 都市中心・高利便性 | 1.10 | 赤れんがテラス |
| 標準的都市型 | 1.00 | 一般的な都市部 |
| アクセスが不便 | 0.90 | 公共交通から遠い |
| 環境マイナス | 0.80 | 騒音あり |

#### Companion（共食係数 ×1.00〜1.10）
LTC事業連動。共食のプラスのみを係数化。

| シーン | 係数 |
|--------|------|
| ウェディング / プロポーズ | 1.10 |
| 顔合わせ / 記念日 | 1.05 |
| 接待ディナー | 1.03 |
| デート（通常） | 1.02 |
| 通常利用 / **一人ディナー** | **1.00** |
| ギフト利用 | 0.95 |

> v3.0変更: 一人ディナー 0.95→1.00。一人で来るゲストを否定しない。

### 2. 期待ギャップスコア B（0.5〜1.5）

```
B = f(料理期待ギャップ, Place期待ギャップ, ブランド期待ギャップ)
```

B > 1.0: 期待を超えた / B = 1.0: 期待通り / B < 1.0: 期待を下回った

**測定プロトコル**: ① 予約時に事前期待を記録 → ② 退店時に3問（料理/場所/ブランド）5段階で期待との差を測定 → ③ 月次で0.5〜1.5にスケーリング → ④ 四半期でトレンド分析

**原則**: A値がブランド認知に先行して成長し続ければ、B値は維持される。

### 3. 満足度スコア C

```
C = A × B
```

A値が美味しさの絶対値、Bが期待との相対差、Cがゲストの体感満足度。

### 4. 並列指標

#### S-Score（サステナビリティ）
```
S = 海洋持続性 + 大地循環性 + 文化継承性
```
SATの原体験（釧路での昆布採り）に根ざすブルーエコノミー指標。

#### P-Score（スタッフの誇り）✦ NEW
```
P = 「今の仕事を、家族に誇りを持って語れますか？」
```
A値が高くてもP-Scoreが低ければ、SATの哲学に反する。

**A値・S-Score・P-Scoreの3指標が揃って初めて、SVDの哲学が実現される。**

## Anti-patterns

- ❌ 「味」だけで美味しさを語ってはいけない（味は全体の一要素）
- ❌ 方程式は全て**掛け算**で構成 — どの要素がゼロでも全体がゼロ
- ❌ 「なんとなく美味しかった」で終わらせない — 必ず数値化する
- ❌ 方程式の前に**哲学**がなければいけない — 道具には目的が必要
- ❌ 期待値に依存し過ぎない — A値の成長が常にBに先行すべき
- ❌ 採点にルーブリックを使わずに自己基準で付けてはいけない
- ❌ 一人で来るゲストを係数で否定してはいけない（Companion最低=1.00）
- ❌ A値だけで判断してはいけない — S-Score, P-Scoreも見よ
